# `retrieval_gold.json` — source-level dataset for testing RAG retrieval

Question → which **documents (source)** should appear in the search results.
Complements `dataset.json` (that one is chunk-level, a single `gold_chunk_id`, generated by an LLM).
This one is **manual and source-level**, with several relevant docs and abstain cases.

## Record format
```json
{
  "id": "single-01",
  "query": "How do I run some code after returning the response...",
  "style": "natural | keyword | identifier",
  "type": "single | multi | abstain",
  "relevant_sources": ["tutorial__background-tasks"],
  "note": "hint describing what this question covers"
}
```
`relevant_sources` — file names without `.md` (= the `source` field in chunk payloads). Order:
most relevant first. For `abstain` the list is empty.

## Question types
| type | count | meaning |
|---|---|---|
| `single`  | 51 | one obvious source — basic retrieval@k check |
| `multi`   | 13 | the question spans several docs — checks COMPLETENESS (whether all relevant ones are in top-k) |
| `abstain` | 4  | out of corpus (k8s, WebSockets, MongoDB, OpenRouter) — the bot should say "I don't know" rather than hallucinate |

The styles (`natural` / `keyword` / `identifier`) mirror the logic in `build_dataset.py`: real
users ask in different ways, which shows where BM25 wins (identifier) vs dense (natural).

## Running the evaluator
```powershell
python eval/evaluate_sources.py                 # hybrid_plus, top-5
python eval/evaluate_sources.py --method dense --k 3
python eval/evaluate_sources.py --collection docs_chunks_nocode
```
Metrics: `any_hit@k` (topic found), `full_cov@k` (all relevant sources in top-k),
`src_prec@k` (noise in the window), `multi full_cov@k` (completeness on multi-doc), `abstain_ok`
(weak top-1 cosine → correct refusal).

## Baseline measurement (hybrid_plus, top-5 sources, collection `docs_chunks`)
- `any_hit@5 = 1.000` — for EVERY answerable question at least one correct source was found.
- `full_cov@5 = 0.952`, `multi full_cov@5 = 0.769` (10/13).
- **Expected multi-doc misses:** a question that requires 5–6 docs (the entire `security/`
  or `dependencies/` section) does not fit into the top-5 *sources*. For those, raise `--k`
  (e.g. `--k 8`) — this is a lack of window slots, not a retrieval defect.
- **Abstain:** 3/4 correctly weak. "Deploy to Kubernetes" gives cos≈0.57 (a LEAK at the
  0.45 threshold) — a useful signal: the refusal threshold must be tuned per domain (like the
  semantic-cache threshold). This is why an abstention prompt plus a threshold is needed, not
  just top-k.
